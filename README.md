## About The Project

Different ways to integrate Swagger and the OpenAPI specification into express server projects. All 4 projects share the same API specification (a simple sample to manage tasks).
1) A mock server generated with Swagger Codegen and express-typescript template (port 8080)
2) A client that uses the SDK generated by Swagger Codegen
3) A design-first approach to create an express server based on the openapi specification (port 3000)
4) A code-first approach to document with the openapi specification an existing API (port 3001)

## Getting Started

### Prerequisites

You will need git, node and docker installed in your computer

### Installation

1. Clone the repo
   ```sh
   git clone git@github.com:samuelcasanova/swagger-sample.git
   ```
2. Import the collection and environments from the postman folder into your postman workspace
3. Run all the servers with docker compose
```sh
   docker compose up
   ```

<p align="right">(<a href="#readme-top">back to top</a>)</p>

### Usage with postman

1. Select the environment to run the tests
2. Use the 'Run Collection' option in postman to run all the requests and tests

### Usage with node and supertest

1. Change to swagger-sdk-client-cli directory
   ```sh
   cd swagger-sdk-client-cli
   ```
2. Install the dependencies
   ```sh
   npm install
   ```
3. Run the tests against the mock server
   ```sh
   npm test
   ```
